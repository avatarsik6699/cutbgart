# Security Policy

Please report suspected vulnerabilities privately through the monitored
[cutbg security contact](https://t.me/+HaqBWI1A3vg4MWJi). Do not include a real
user image, private disclosure details in a public issue, credentials, tokens,
or destructive proof-of-concept traffic.

Include the affected URL or component, impact, reproducible steps using a
synthetic fixture, and any safe mitigation you already identified. The
maintainer will acknowledge a credible report within 3 business days, assign
severity and coordinate remediation privately. Public disclosure is
coordinated after a fix is deployed, or on an explicitly agreed date.

Supported security updates apply to the current production release. See
`/.well-known/security.txt` for the canonical contact and expiry.
